# **Enhanced Real Estate Matchmaking Algorithm**

---

## **1️⃣ Overview**

**Goal:** Score compatibility between a real estate project/company (founder) and an investor, producing a **0–100% match score** with explainable factors.

**Key Improvements over a simple weighted model:**

1. **Semantic Thesis Matching:** Captures investor intent and project characteristics beyond static labels.
2. **Conditional / Interaction Weights:** Stage, check size, and property type interact non-linearly.
3. **Deal Structure Compatibility:** Auto-penalty for mismatched debt/equity preferences.
4. **Behavioral Layer:** Tracks investor activity, response history, deal frequency.
5. **Economic Reality Layer:** Ensures investor can actually participate in the deal.

## **3️⃣ Enhanced Matching Components**

### **A. Hard Constraints (Pre-Filter)**

* Check size overlap ≥ 50% → otherwise reject
* Stage compatibility ≥ threshold → otherwise reject
* Regulatory / market exclusions → auto-reject
* Investor active in last 24 months → otherwise penalty

---

### **B. Semantic Fit (Property + Thesis Embeddings)**

**Process:**

1. Encode project description and investor thesis using **vector embeddings** (OpenAI embeddings, or custom BERT/LegalTech embeddings for real estate).
2. Compute **cosine similarity** → score 0–1
3. Multiply by **weight factor (30%)**

**Example:**

* Investor: “Focus on Bay Area multifamily development projects, prefers stabilized assets or bridge financing.”
* Project: “Raising $3M for Bay Area multifamily construction project, bridge loan structure.”
* Semantic similarity = 0.92 → contributes 27.6 points (0.92 × 0.30 × 100)

---

### **C. Stage / Deal Type Compatibility**

| Stage            | Investor Preferred Stage | Score  |
| ---------------- | ------------------------ | ------ |
| Pre-Development  | Pre-Dev / Bridge         | 100%   |
| Construction     | Construction / Bridge    | 100%   |
| Stabilized Asset | Stabilized / REIT        | 100%   |
| Adjacent stage   | One level difference     | 70–85% |
| Non-adjacent     | >1 stage                 | 0–50%  |

* Weight: 25%
* Conditional multipliers: geography more important in early stages, less in stabilized assets.

---

### **D. Geography Matching**

**Logic:**

* Expand city → metro → state → country
* Global investors → 100%
* Regional investors → penalty if project outside coverage

Weight: 20%

Example Multiplier Table:

* Exact city match: ×1.0
* Metro / region match: ×0.9
* Country match: ×0.8
* No match: ×0.5

---

### **E. Check Size / Ticket Size Matching**

**Algorithm:**

```
Overlap Ratio = min(project_max, investor_max) - max(project_min, investor_min) 
                -----------------------------------------------
                max(project_max, investor_max) - min(project_min, investor_min)
```

* Result: 0–1
* Weight: 15%
* Auto-reject if overlap < 25%

---

### **F. Investor Type Compatibility**

| Investor Type  | Preferred Deal Type / Stage                 | Score |
| -------------- | ------------------------------------------- | ----- |
| VC             | Early-stage development, equity             | 100%  |
| Family Office  | Bridge / Development / Stabilized, flexible | 100%  |
| PE / Debt Fund | Stabilized / Bridge, larger ticket          | 100%  |
| REIT           | Stabilized assets, equity / joint venture   | 100%  |
| Misaligned     | Stage/type mismatch                         | 0–50% |

Weight: 10%

---

### **G. Activity & Behavior Layer**

Optional multiplier:

* Active deal in last 12 months: ×1.1
* No deals last 2 years: ×0.7
* Avg response rate > 70%: ×1.05

---

### **H. Deal Structure Compatibility**

* If project offers equity but investor prefers debt → penalty ×0.6
* Revenue participation mismatch → penalty ×0.8
* Fully compatible → ×1.0

---

## **4️⃣ Final Score Formula**

```
Final Score = (
    Semantic Fit × 0.30 +
    Stage Compatibility × 0.25 +
    Geography Compatibility × 0.20 +
    Check Size Overlap × 0.15 +
    Investor Type Compatibility × 0.10
) × Deal Structure Multiplier × Activity Multiplier
```

* Output: **0–100%**
* Thresholds:

  * ≥70% → strong match
  * 50–69% → moderate match
  * <50% → weak match / optional display
